## Challenges & Improvements 🔄

### 控制条状态管理过于复杂

**问题描述：**
在视频切换时，希望控制条不出现，但改了很多次才成功：
- 第一次改：每次切换都会出现
- 后面又改了几次：还是会在某些情况下出现
- 最后才搞定

**根本原因分析：**

| 原因 | 说明 |
|------|------|
| **状态碎片化** | 控制条显示受多个状态影响：播放状态、切换状态、用户交互、计时器等 |
| **缺乏明确规则** | 没有清晰的"控制条显示规则表"，各场景下的行为不明确 |
| **逻辑分散** | 控制条可见性的逻辑分散在多个地方 |
| **调试困难** | 难以追踪是哪个条件导致当前行为 |

**状态空间示例：**
```dart
// 控制条显示可能受这些因素影响：
❌ _controller.state.isPlaying
❌ _controller.state.isBuffering
❌ _isSwitchingVideo (切换状态)
❌ _userInteracted (用户最近交互)
❌ _autoHideTimer (自动隐藏计时器)
❌ 视频首次加载
❌ 错误状态
```

**改进建议：**

1. **使用单一状态变量**
```dart
enum ControlBarVisibility {
  visible,   // 明确显示
  hidden,    // 明确隐藏
  transitioning, // 过渡中
}

// 单一状态决定
ControlBarVisibility visibility = _computeVisibility();
```

2. **提前定义状态规则表**
```dart
ControlBarVisibility _computeVisibility() {
  if (_isSwitchingVideo) return ControlBarVisibility.hidden;
  if (_userRecentlyInteracted) return ControlBarVisibility.visible;
  if (_isPlaying && _autoHideTimerExpired) return ControlBarVisibility.hidden;
  return ControlBarVisibility.visible;
}
```

## Key Insights

1. **复杂 UI 状态应该由单一状态变量控制** - 控制条、全屏、弹幕等组件都应该有明确的状态管理
2. **提前定义清晰的状态规则表** - 任何有多状态的 UI 组件都应该有明确的规则
3. **集中管理状态变化逻辑** - 避免逻辑分散在代码各处
4. **流程改进持续有效** - Epic 0 的改进在 Epic 6 中发挥了作用
